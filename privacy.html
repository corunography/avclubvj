Privacy Policy

Last updated: April 18, 2026

Phasion Live is a live visualizer application used by DJs and venues. This policy has two parts: how audience data is handled when you interact with the app via the submission page (QR code), and how the operator's desktop app handles crash telemetry. Audiences are only affected by the first part.

What We Collect

When you use the audience submission page, the following data may be collected:

  • Text messages you submit (up to 240 characters)
  • Photos you upload (resized to fit the display)
  • AI prompts you submit for visual generation
  • Team names you enter for trivia games
  • Poll votes you submit
  • Username if you optionally provide one

All submissions are voluntary. No account, login, or personal identification is required.

How Data Is Stored

  • Submissions are temporarily stored using Cloudflare R2 (cloud object storage) to relay content from your phone to the venue's display.
  • Content is automatically deleted after the venue's app retrieves it, typically within seconds.
  • Photos may be saved to the DJ's local machine for display during the event. These are not uploaded to any external server.

What We Do NOT Do

  • We do not sell, share, or monetize your data.
  • We do not create user profiles or track you across sessions.
  • We do not use advertising trackers or analytics cookies.
  • We do not store your IP address beyond what Cloudflare retains in standard server logs.
  • We do not retain your submissions after they have been delivered to the venue's display.

Third-Party Services

  • Cloudflare provides the network infrastructure (Workers, R2 storage, and Tunnel). Cloudflare may process IP addresses and request metadata per their own privacy policy.
  • Apple ShazamKit may receive audio fingerprints when the DJ enables song identification. No audio recordings are stored.
  • QR Server API generates QR code images. Only the URL is sent; no personal data.
  • Sentry receives anonymous crash reports from the desktop app only (never from the audience page). Details in the "Operator Desktop App — Crash Telemetry" section below. See Sentry's privacy policy.

Public Display

By submitting content through the audience page, you acknowledge that your message, photo, or prompt may be displayed publicly on a screen at the venue. The DJ or venue operator has moderation controls and may approve, reject, or skip any submission.

Data Retention

  • Audience submissions: deleted from cloud storage after retrieval (typically seconds).
  • Photo history: stored locally on the DJ's machine only. Not synced to any cloud service.
  • Trivia scores: stored locally on the DJ's machine for the duration of the event.
  • Usage statistics: anonymous aggregate counts (number of messages, photos per day) stored for up to 90 days for monitoring purposes. No personal data is included.

Children

Phasion Live is intended for use at live entertainment venues. We do not knowingly collect information from children under 13. If a venue operator becomes aware that a child has submitted content, they should use the moderation tools to reject it.

Operator Desktop App — Crash Telemetry

This section applies only to DJs and venue operators who install and run the desktop app. Audiences submitting via the QR code are not affected.

When the desktop app encounters an unexpected error, an anonymous crash report is sent to Sentry to help the developer diagnose and fix bugs. Reports contain:

  • The error message and stack trace
  • App version, OS name and version, Electron / Chrome / Node.js versions
  • A breadcrumb log of the last ~100 state transitions (e.g. "tunnel connected", "Syphon started", "moderation toggled") — no submission content is included
  • A random anonymous venue ID (e.g. avclubvj-a1b2c3d4) so multiple reports from the same installation can be grouped together. This ID is generated on first run and stored locally; it is not linked to your identity, email, account, or IP address.

Reports do not include audience submissions, passwords, authentication tokens, network URLs, file paths outside short log snippets, usernames, or IP addresses. Before transmission, error messages and stack traces pass through a scrubber that redacts email addresses, IP addresses, bearer tokens in URLs, and /Users/<name> paths as a defense-in-depth measure.

Legal basis (GDPR Art. 6(1)(f)): legitimate interest in diagnosing software defects and maintaining application stability. You may opt out at any time via Settings → Privacy → Share crash reports in the desktop app. Disabling takes effect after the next app restart. Reports already sent cannot be recalled.
